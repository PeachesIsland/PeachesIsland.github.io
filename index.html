什么是redis？

redis 是一种基于内存的数据库，对数据的读写操作都是在内存中完成。

因此读写速度非常快，常用于缓存，消息队列，分布式锁等场景。

redis和memcache有什么区别？

共同点：

都是基于内存的数据库，一般都用来当作缓存使用

都有过期策略

两者的性能都很高

区别：

数据类型：redis支持的数据类型更丰富，memcache只支持最简单的额key-value数据类型

持久化：redis支持持久化，memcache不支持

集群模式：redis原生支持集权模式，memcached没有原生的集群模式，需要依赖客户端来实现往集群中分片写入数据

其他：redis支持 发布订阅模式，lua脚本，事务等功能，memcached不支持

mysql 的锁是什么？

mysql 的锁是一种机制。用来保证并发操作/访问数据时的数据完整性，一致性，正确性。

mysql 利用锁机制创建了不同的事务隔离级别

mysql 的唆的触发和释放是由mysql 自动管理的，而具体的锁机制和行为取决于所使用的存储引擎，事务隔离级别以及执行的语句类型。

mysql 锁的分类？

按锁的粒度划分： 表级锁，行级锁，页级=锁

按锁级别划分：共享锁（Shared Lock：select），排他锁（Exclusive Lock，操作：insert，update，delete）

按加锁方式划分：自动锁，显式锁

按操作划分：DML锁，DDL锁

按使用方式划分：乐观锁，悲观锁

mysql 在什么情况下会触发锁？

  1. 事务操作
    • 事务开始时，会根据事务的隔离级别和存储引擎选择对应的锁（行级锁，表级锁，或其他级别的锁）来保护所设计的数据。
  2. 显式锁定
    • 使用 lock tables 锁定一个或多个表
    • 使用 select … for update 或 select … lock in share mode 语句 ，在查询中获得行级锁
  3. 自动锁定
    • 如执行insert update selete 等修改数据的语句，mysql会在执行期间自动获取合适的锁
  4. 冲突锁定
    • 多个事务同时对统一数据进行修改时，可能会发生锁冲突。

  2. 使用url回写来实现

Session的一种方式。

GET请求和POST请求的区别是什么？

  1. 使用场景 GET用来获取资源 POST用来传输实体主体
  2. 参数 GET在 url 中，post 在 实体中
  3. 安全性 get 是安全的，不会改变服务器状态，post 是不安全的
  4. 幂等性 get 是幂等的，post 不是
  5. 可缓存 get 是可缓存的，可被存储的 post 不可以

http是什么？

发表于 2023-07-05 分类于 http

http是什么？

http 是超文本传输协议 ，是一个应用层的协议

主要特点：

  1. 支持客户/服务器模式
  2. 简单快速
  3. 灵活
  4. 无连接，http1.0 使用短链接，http1.1（目前主流） 起默认使用长连接（keep-alive）。
  5. 无状态

http请求结构有哪些？

  1. 请求行（Request Line），包含请求方法、请求的URL和HTTP协议的版本。
  2. 请求头（Request Headers）包含一系列的键值对，用于描述请求的各种属性和要求。常见的请求头包括：
  • Host：指定服务器的域名或IP地址。
  • User-Agent：标识发送请求的用户代理（通常是浏览器）。
  • Accept：指定可接受的响应内容的MIME类型。
  • Content-Type：指定请求体的MIME类型（仅适用于POST、PUT等包含请求体的方法）。
  • Authorization：用于身份验证的凭证信息
  1. 空行：请求头和请求体之间必须有一个空行。
  2. 请求体（Request Body）：可选项，包含发送给服务器的数据，通常用于POST、PUT等需要提交数据的请求方法

http响应报文有哪些？

  1. 状态行（Status Line） 包含HTTP协议版本、状态码和对应的状态信息，示例：HTTP/1.1 200 OK
  2. 响应头（Response Headers）
  3. 空行：响应头和响应体之间必须有一个空行。
  4. 响应体（Response Body）

常见http状态码有哪些？

200：ok，正常返回信息

400：客户端请求语法错误

403：重定向

404：客户端请求的资源不存在

500：服务器错误

503：服务器错误
